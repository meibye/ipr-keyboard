---
name: design-rfc
description: Produce or update a design note or RFC covering problem framing, options, tradeoffs, risks, and implementation approach.
---

# When to use
- Before major feature work
- Before architectural shifts
- Before migrations
- Before significant operational changes

# Inputs
- problem statement
- constraints
- candidate approaches
- affected systems

# Procedure
1. Define the problem and success criteria.
2. Capture constraints and assumptions.
3. Compare options.
4. Recommend an approach with tradeoffs.
5. Outline implementation, rollout, and validation.
6. Identify open questions and risks.

# Quality bar
- Be explicit about tradeoffs
- Distinguish decisions from open questions
- Avoid premature low-level implementation detail

# Output format
- context
- options
- decision
- consequences
- rollout plan
